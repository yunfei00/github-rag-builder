# ROADMAP.md

## 项目名称
近场扫描系统（Near Field Scan System）

## 当前目标
从零开始构建一个基于 Python 3.11 + PySide6 的桌面应用。
当前只做“可持续迭代的工程化骨架”，不实现真实硬件业务。

---

## 开发总原则
1. 每次只完成一个小阶段
2. 每次改动都应能运行或至少能被验证
3. 优先保证目录结构、模块边界、命名清晰
4. 先占位，后细化
5. 不要一次性实现过多功能
6. 所有功能开发都围绕可扩展性展开
7. UI 优先中文
8. 后续所有硬件、协议、数据处理逻辑都要模块化

---

## Phase 1：项目骨架与基础 UI
### 目标
建立最小可运行的 PySide6 工程。

### 任务
- 创建 Python 包 `nfs_scanner`
- 创建程序入口 `python -m nfs_scanner.main`
- 创建主窗口
- 建立基础 UI 分区：
  - 左侧控制面板
  - 中间主视图
  - 右侧设置面板
  - 底部日志区
  - 底部状态栏
- 完成 README 和 requirements.txt

### 验收标准
- 项目可以启动
- 主窗口可以正常显示
- UI 分区明确
- 无真实设备逻辑

---

## Phase 2：UI 组件拆分
### 目标
将主窗口拆分为独立面板组件，便于后续开发。

### 任务
- ControlsPanel
- HeatmapView
- SpectrumPanel
- LogPanel
- MainWindow 负责组装，不堆积太多逻辑

### 验收标准
- 每个 UI 组件独立文件
- MainWindow 代码保持清晰
- 各区块支持后续扩展

---

## Phase 3：应用层骨架
### 目标
创建应用服务层和基础模型层。

### 任务
- DeviceManager
- ScanManager
- AppConfig / 基础数据模型
- logging/config 占位实现

### 验收标准
- core / infra 目录可导入
- 不为空文件
- 为后续真实逻辑预留接口

---

## Phase 4：运动控制模块骨架
### 目标
先搭建运动控制的接口与 UI，不接真实硬件。

### 任务
- 定义运动控制抽象接口
- 创建串口配置 UI 占位
- 创建手动移动 UI 占位
- 创建连接/断开状态占位逻辑

### 验收标准
- 有明确接口边界
- UI 和设备逻辑分层
- 不接真实串口

---

## Phase 5：扫描任务模块骨架
### 目标
建立扫描参数和任务执行框架。

### 任务
- 扫描参数数据模型
- 扫描任务配置 UI
- 扫描开始/停止占位逻辑
- 日志联动

### 验收标准
- 可以从 UI 收集参数
- 有清晰的数据结构
- 不实现真实扫描执行

---

## Phase 6：热力图与数据视图骨架
### 目标
建立主视图区域的后续扩展基础。

### 任务
- 主显示区域组件化
- 预留图片加载接口
- 预留热力图图层接口
- 预留缩放/平移能力

### 验收标准
- HeatmapView 结构清晰
- 便于后续叠加显示

---

## 当前明确不做
- 真实串口通信
- 真实频谱仪接入
- 真实相机接入
- 数据库存储
- 授权系统
- 打包发布
- 复杂算法逻辑

---

## Codex 执行规则
1. 每次只完成当前指定的一个 phase 或一个子任务
2. 不要跳阶段实现功能
3. 不要擅自引入重型依赖
4. 完成后必须说明：
   - 修改了哪些文件
   - 如何运行
   - 下一步建议
