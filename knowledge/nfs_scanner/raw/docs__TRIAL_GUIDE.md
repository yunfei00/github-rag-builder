# V0.1 Trial Guide

## Start the App

1. Install dependencies:

```powershell
pip install -r requirements.txt
```

2. Start the application from the project root:

```powershell
python -m nfs_scanner.main
```

If you are using the Windows Python launcher, you can also run:

```powershell
py -3.11 -m nfs_scanner.main
```

## Run a Mock Scan

1. Open the app.
2. In the left-side `扫描参数` area, keep the default values or enter a small test range such as:
   - `起始X = 0`
   - `终止X = 4`
   - `步长X = 1`
   - `起始Y = 0`
   - `终止Y = 4`
   - `步长Y = 1`
   - `扫描模式 = snake`
3. Click `开始扫描`.
4. Watch the bottom area for:
   - current `Job ID`
   - `Status`
   - `Progress`
5. Watch the center heatmap view update during the scan.

If you want to restore the default scan settings, click `恢复默认配置`.

## What to Expect

- The scan is a mock/demo scan only.
- Motion, spectrum, and camera data are generated by mock devices.
- The center heatmap updates progressively while the scan runs.
- The bottom log area records scan progress and a final scan summary.
- When the scan finishes, the app automatically saves the latest dataset.

## Output Files

The latest mock scan is saved automatically to:

```text
./output/latest_scan/
```

Expected files:

- `scan_config.json`
- `positions.npy`
- `spectrum.npy`
- `images.npy`

If you run the **扫描控制页面** (serial motion path scan), it now also writes:

- `scan_plan_points.csv` (planned path points)
- `scan_executed_points.csv` (points already dispatched to the controller)
- `scan_execution_status.json` (completed flag and point counters)

The folder is created automatically if it does not already exist.

## Current Limitations

- No real hardware communication is implemented.
- The scan runs in the UI thread, so very large scans are not recommended for trial testing.
- The `latest_scan` folder is reused for each new scan.
- Some controls are still placeholders for future development.
